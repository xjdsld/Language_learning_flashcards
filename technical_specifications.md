Проєкт: Консольний застосунок для вивчення слів (Flashcards App)
Мова програмування: Python
База даних: SQLite
Архітектура: Модульна

1. Загальний опис системи

Застосунок призначений для:

- збереження англійських слів у базі даних

- автоматичного перекладу слова українською мовою

- створення навчальних карток (flashcards)

- запуску простої навчальної сесії через консольний інтерфейс

Архітектура розділена на окремі модулі:

- database.py
- logging.py
- main.py
- session.py
- translation_service.py

2. Архітектура системи
Основні компоненти:
- main.py	Точка входу, логіка меню
- database.py	Робота з SQLite
- session.py	Модель даних (Card)
- translation_service.py	API перекладу
- logging.py	Конфігурація логування
3. Опис модулів
3.1 database.py
Призначення
Забезпечує роботу з базою даних SQLite.
Клас: Database
__init__(self)

Створює підключення до database.db

Ініціалізує cursor

Логує ініціалізацію

- create_db(self)
Створює таблицю words, якщо вона не існує.

Структура таблиці:
id	INTEGER PRIMARY KEY AUTOINCREMENT
word	TEXT
translation	TEXT
sentence	TEXT

- create_card(self)

Отримує перший запис із таблиці

Якщо запис відсутній — повертає None

Якщо запис є — повертає об'єкт Card

Return:
Card | None

- add_card(self, card)
Додає нову картку в базу даних.
Параметри:

card: Card

Використовується параметризований SQL-запит для запобігання SQL-injection.

- get_all_cards(self)
Повертає всі записи з таблиці.
Return:
list[tuple]

3.2 session.py
Призначення

Містить модель даних картки.

Клас: Card
Атрибути:

word: str

translation: str

sentence: str

3.3 translation_service.py
Призначення

Здійснює HTTP-запит до API перекладу.

Клас: Translation
Метод: translation(self, word)

Формує GET-запит

Отримує JSON-відповідь

Повертає переклад:

data["responseData"]["translatedText"]
Використані бібліотеки:

requests

logging

3.4 logging.py
Призначення

Конфігурація логування для всього застосунку.

Використовується:

logging.basicConfig

Рівень логування: INFO

Формат:

%(asctime)s - %(name)s - %(levelname)s - %(message)s
3.5 main.py
Призначення

Головна точка входу.

Основна функція: menu()
Логіка:

Ініціалізація:

Database

створення таблиці

Translation

Виведення меню:

1 — навчальна сесія

2 — додати слово

Обробка вибору користувача

Сценарій 1 — Навчальна сесія

Отримується картка з БД

Показується слово

Після Enter показується переклад і приклад

Сценарій 2 — Додавання слова

Користувач вводить слово

Виконується автоматичний переклад

Користувач вводить приклад

Картка зберігається в БД

4. Потік виконання
main.py
   ↓
Database()
   ↓
Translation()
   ↓
Користувач обирає дію
   ↓
Або створюється картка
Або додається нове слово

5. Залежності

Python 3.x

sqlite3 (вбудований)

requests

6. Логування

Логуються такі події:

- створення БД

- вибір користувача

- створення картки

- додавання слова

- помилки SQLite

- некоректний ввід

7. Обробка помилок

ValueError при некоректному вводі числа

sqlite3.Error при помилках БД

8. Висновок

- модульну архітектуру

- базову роботу з SQLite

- використання HTTP API

- використання logging

- базову обробку винятків
